The purpose of Re-upload Data Table is to let you update a data table's contents (e.g., because you've added some new rows or edited some cell values) without having to enter the properties of the data table's columns all over again.

Moreover, if you later use Export ezEML Data Package or Send to EDI, which creates a zip file containing the EML metadata and all of the data tables and other entities associated with the data package, it is essential that the final, corrected versions of all files have been uploaded to ezEML, because it is the versions that have been uploaded to ezEML that will be included in the exported data package.

Re-upload Data Table requires the number of columns in the table to be the same as in the original table, but it allows column names and types to change.
