The purpose of Re-upload Data Table is to let you update a data table's contents (e.g., because you've added some new rows or edited some cell values) without having to enter the properties of the data table's attributes all over again. It assumes the properties of the data table's attributes are largely unchanged.

To be more specific, it is OK to use Re-upload Data Table if the number of rows in the table have changed or if values in table cells have changed, but the column types (categorical, datetime, numerical, or text) should not have changed and no columns should have been added, deleted, or moved.

It is OK if some column names in the header row have changed, and it is OK if some codes for Categorical variables have been added or deleted.

If the data table has changed in ways that are not allowed, click Cancel below and load the table anew using Load Data Table from CSV File. {# , and then clone attributes from the existing version of the table, as needed.#}
